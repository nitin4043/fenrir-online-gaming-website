Scope of Work

The freelancer must build a full-stack online gaming website prototype for ArcadeHub Nexus. The application must include a React-based frontend, Node.js and Express backend, local database setup, authentication, seeded sample data, and clear local run instructions. The website must run on a local machine without requiring a cloud database, paid API service, private API key, payment gateway, or deployment platform. SQLite is the preferred database because it supports local testing, though another local database may be used if the setup is documented.

The frontend must include the main user-facing screens described in instructions.md. These include a homepage, sign-up page, login page, games catalog page, game detail page, leaderboard page, and profile dashboard page. The homepage must present ArcadeHub Nexus branding, the main product message, a Browse Games action, a Sign In action, featured games, leaderboard preview, and daily challenge preview. The catalog must load game records from the backend API and support title search, category filtering, rating sorting, and an empty state when no results are found. Each game detail view must show the selected game information, instructions, a Play Now button, a demo preview area, a favorite button, and a score submission form.

The backend must provide Express API routes for authentication, games, favorites, scores, leaderboard, and health checking. Authentication must include signup, login, JWT creation, password hashing, and a protected profile endpoint. Game data must come from the database through API routes rather than being hardcoded only in frontend components. Favorites must be saved through backend routes, and submitted scores must be stored so they can appear in leaderboard-related data. The database must include users, games, favorites, and scores tables, with seed data for at least 12 games, 10 leaderboard score records, and 1 demo user.

The website must follow the dark arcade visual direction shown in reference_image.png and assets.pdf. The interface should use a deep navy background, neon cyan primary accent, electric violet secondary accent, gold leaderboard highlight, card-based sections, rounded panels, and one consistent sans-serif font family. The layout must be responsive at desktop, tablet, and mobile widths, with no horizontal overflow at 375px. The submitted project must include screenshots for desktop and mobile views.

The final delivery described by this task must include the complete source code, frontend folder, backend folder, database setup files, seed script, README.md, screenshots folder, and any local environment example file needed to run the project. README.md must list the tech stack, folder structure, frontend setup command, backend setup command, database setup command, seed command, frontend run command, backend run command, demo login credentials, API route list, and out-of-scope notes. The project must not include gambling, betting, casino mechanics, real-money payments, crypto wallets, live multiplayer, admin dashboard, cloud-only database, copyrighted game art, or real gaming company branding.
